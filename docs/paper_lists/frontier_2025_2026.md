# 2025-2026 Conference Frontier Digest

Last curated: 2026-05-09

This page is the cross-venue update layer for the atlas. It is intentionally selective: a paper belongs here only if it changes how a reader should think about a roadmap, benchmark, dataset, or build path.

## How to read this digest

Use this page before opening the per-venue lists when your goal is field orientation:

1. Start with the trend matrix.
2. Pick the papers that match your current system layer.
3. Jump back to the roadmap page for context.
4. Use the venue pages when you need a wider list.

## High-priority reading queue

### If you track VLA systems

1. [Embodied-R1](https://embodied-r1.github.io/) - reasoning and pointing as an embodied intermediate interface.
2. [MemoryVLA](https://shihao1895.github.io/MemoryVLA/) - memory bank for temporally aware manipulation.
3. [WholeBodyVLA](https://github.com/OpenDriveLab/WholebodyVLA) - whole-body VLA for loco-manipulation.
4. [Long-VLA](https://long-vla.github.io/) - long-horizon task decomposition and phase-aware training.
5. [RoboInter](https://github.com/InternRobotics/RoboInter) - intermediate representation suite for plan-then-execute manipulation.

### If you track world models and sim-to-real

1. [DreamZero](https://dreamzero0.github.io/) - WAM framing where future world-state prediction and action generation support zero-shot policy behavior.
2. [Fast-WAM](https://yuantianyuan01.github.io/FastWAM/) - tests whether WAMs need explicit future imagination at inference or mainly benefit from video co-training.
3. [GigaWorld-Policy](https://arxiv.org/abs/2603.17240) - action-centered WAM that keeps video generation as an optional auxiliary signal for faster deployment.
4. [ParticleFormer](https://proceedings.mlr.press/v305/huang25c.html) - 3D point-cloud world model for mixed-material manipulation.
5. [PIN-WM](https://www.roboticsproceedings.org/rss21/p153.pdf) - physics-informed world models for non-prehensile manipulation.
6. [World Model Implanting](https://proceedings.mlr.press/v267/yoo25a.html) - test-time adaptation through world-model insertion.
7. [AgentWorld](https://yizhengzhang1.github.io/agent_world/) - simulation platform for household mobile manipulation.
8. [DynScene](https://openaccess.thecvf.com/content/CVPR2025/html/Lee_DynScene_Scalable_Generation_of_Dynamic_Robotic_Manipulation_Scenes_for_Embodied_CVPR_2025_paper.html) - scalable dynamic scene generation for robot learning.

### If you track affordance, grasping, and manipulation

1. [SeqAfford](https://seq-afford.github.io/) - sequential 3D affordance reasoning with MLLMs.
2. [Robotic Visual Instruction](https://openaccess.thecvf.com/content/CVPR2025/html/Li_Robotic_Visual_Instruction_CVPR_2025_paper.html) - visual instruction interface for spatially precise robot guidance.
3. [Dex1B](https://www.roboticsproceedings.org/rss21/p106.html) - billion-scale dexterous demonstrations.
4. [DexUMI](https://proceedings.mlr.press/v305/xu25b.html) - human hand as a universal manipulation interface.
5. [BiAssemble](https://proceedings.mlr.press/v267/shen25i.html) - collaborative bimanual affordance for geometric assembly.

## Maintenance rule

Do not add every accepted paper here. Add a paper only when it clarifies one of these repository questions:

- What is the action interface?
- Where does state or memory live?
- How does the system improve after failure?
- What changes when embodiment, scene, or horizon changes?
- Does it create a benchmark, dataset, simulator, or open tool that a reader can use?
