# Maintenance Guide

This file turns the atlas from a one-time curation pass into a maintainable repository.

## Cadence

### Monthly

- review `frontier-2025-2026` entries in roadmap pages
- check whether one or two new releases materially change a build path
- refresh company watch items in `docs/maps/industry.md`

### Quarterly

- review internal and external links
- review whether roadmap boundaries still match the active field structure
- prune stale or weak entries that no longer deserve top-level visibility

## Editorial checks

Before adding new material, ask:

1. Is this `classical`, `frontier-2025-2026`, `open-source`, `benchmark`, `simulator`, `industry-system`, or `build-path` relevant?
2. Does it improve a guide page or only inflate a list page?
3. Does it help readers build, compare, or track the field?

## Priority order

Guide pages are canonical. List pages are secondary.

The intended maintenance priority is:

1. roadmap pages
2. maps
3. benchmarks, datasets, frameworks, simulators
4. build paths
5. paper lists

## Link checking rule

When time is limited, verify internal links first, then official project pages, then lower-priority secondary links.
